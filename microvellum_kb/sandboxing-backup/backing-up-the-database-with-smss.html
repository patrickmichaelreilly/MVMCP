SQL Troubleshooting: Backing up the Database with SMSS

Backing up the database in SQL must be done manually because the scheduling feature is not enabled in the express version. 

To run the backup manually, Right-click on the database that is being backed up and go to Tasks | Backup…

The only area in the backup database screen that needs to be verified is the Destination of the files.  The default path is entered automatically.  It may be removed by selecting the Remove button.  Select the Add button to enter a new path for your database to be backed up to.

When the settings are as desired, select OK, and the backup will start.

 


The Microvellum Data folder should be backed up as well.  Here is how that would be done.

Information on System used in sample paths:

Name of Library configuration:  

Advanced_Frameless_Library_English_v036_1

Database Type:  

SQLCE

Files To Be Backed Up:

- Microvellum Data Folder
Backup of Microvellum Data:
  • Find your path to your most recently used Microvellum Data by clicking on Help-> Browse to Microvellum Data
  • Go up one level in the folder structure and copy entire Microvellum Data Folder (e.g. C:\Users\%userprofile%\AppData\Local\Microvellum\Common Microvellum Data\Advanced_Frameless_Library_English_v036_1)
  • Paste Library folder into a backup folder.
