Overview & Tutorials: Formula Generated Materials in Doors

This video explains and demonstrates how to use formula generated materials in the default 048 library data. Prior to this library version, this had not been included in the shipping library data although some Users have been doing this in their company data.
  1. Overview of the reasons why we need these new formula-driven materials.  (begin – 00:49)
  2. Description of synchronization between database and spreadsheet  (begin – 03:40)
    3. Using the ‘deleted’ keyword in the spreadsheet to keep the database & spreadsheet in sync. It is possible to quickly add materials in the spreadsheet environment.
  3. Dynamically modifying material name and thickness.  (begin – 12:35)
    4. Cannot currently add a formula in the program material User Interface. Instead, we must find the material in the spreadsheet and create the formula there.  Use the ISERROR function to prevent the evaluation if there is an error. 
  4. Using formula-driven materials with pointers. (begin – 18:28)
  5. Project Wizard variables that control names & thicknesses of door parts. (begin – 20:21)
  6. Door subassemblies – local variables, part thickness, lookup tables. (begin – 23:10)
  7. Effect of formula-driven materials on part thickness.  (begin – 27:14)
  8. Formula generated materials in action. (begin 30:16)
