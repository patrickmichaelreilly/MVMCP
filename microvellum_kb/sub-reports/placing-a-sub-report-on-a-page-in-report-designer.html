Placing a Sub Report on a Page in Report Designer

The articles in this section of the Knowledge Base involve working with Sub Reports for display in your reports. To learn more about Sub Reports, and how to access this section within the Report Designer, see Understanding Sub Reports in Report Designer. This article provides an overview of inserting a Sub Report on a page.

The Sub Report component cab be placed on any part of a page. The width of the nested page depends on the width of the Sub Report component. See Figure 01 for an example of the Sub Report component and nested page:


Fig. 01 – Sub Report Component and Nested Page

The Can Grow property of the Sub Report component is set to true by default but, when this component is placed, it cannot grow by height. So you should take into account the height of the component on the nested page: it should not be higher than the Sub Report component. When rendering a report, the Sub Report component is rendered as the report page item, the reporting tool renders all sub reports and places them in the container of the Sub Report component.
