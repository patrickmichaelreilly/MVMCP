Working in the Sub Reports Editor of Report Designer

The articles in this section of the Knowledge Base involve working with Sub Reports for display in your reports. To learn more about Sub Reports, and how to access this section within the Report Designer, see Understanding Sub Reports in Report Designer. The purpose of this article is to break down the Sub Reports Editor.

From the Sub Reports Editor, you can configure the settings for the Sub Report and specify the resource.

To open the Editor, double-click on the Sub Report component, as seen in Figure 01.


Fig. 01 – Sub Report Component


Fig. 02 – Sub Report Editor

  1. This is the Settings panel of the Sub Report Editor. When the Page tab is selected, all of the report template pages (except Page1) are displayed here. When rendering the report, it is displayed on this component.
  2. Page – In the Page tab, you can select the report template page to use as the resource for the Sub Report component.
  3. File – In the File tab, you can specify a path to an external report that acts as the resource for the Sub Report component.
    4. This is not necessary if you are building the sub report within the same session of Report Designer.
  4. Hyperlink – In the Hyperlink tab, you can specify a link to an external report that acts as the resource for the Sub Report component.
    5. This is not necessary if you are building the sub report within the same session of Report Designer.
  5. Parameters - In the Parameters tab, you can create and configure the settings that are passed to the sub-report. 

Parameters are used to transfer information, or filter data, to the sub report. To create a parameter, you should:

  1. Select the Add button in the Parameters tab. 
  2. Specify the name of the parameter and its expression. In the expression, you have the option to specify:
    1. The Data Column
    2. Variable
    3. Any other expression

After a parameter is setup, go to the resource for the sub report component, whether it is a page or another report, and specify this parameter, for example, in the filter expression.
