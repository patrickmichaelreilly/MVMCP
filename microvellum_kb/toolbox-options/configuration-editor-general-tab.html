Configuration Editor: General Options

This article provides an overview of the options and settings available on the General tab of the Configuration Editor (Options) Interface.  For a complete list of available tabs and options visit Overview: Configuration Editor (Options) Interface.

Numeric Units and Display

Accuracy – Sets the accuracy that displays in the spreadsheet and reports. If you are using an imperial configuration, you need to set this value to the precision of decimals or fractions. If you are using a metric configuration, enter in a value for how many numbers to display after the decimal.

Fractions/Decimals Radio Button – Selects how the spreadsheet, prompts interface, and reports display numeric values.

Convert to Metric – When using an imperial configuration, the convert to a metric checkbox will convert the reports to metric values when processing the work order. To convert g-code to metric values, use the setting within the tool file. 


Item Number Setup

Sequence – Determines whether an item number increments by a whole number (1.00, 2.00, 3.00…) or decimal (1.01, 1.02, 1.03…).

Increment By – Determines whether item numbers increment across an entire project or if the sequence is restarted for every new room in the project.


Edgebanding for Part Labels

The Edgebanding for Part Labels setting controls edgebanding representation in the part picture of reports and labels.  Edgebanding can be designated either as a thick line or with an X on the line.


Metafile Part Origin

Controls the Part Origin representation in the part picture of reports and labels.


Project Viewer

Default Name of Project Locations – Enter the default name used in the interface for project locations.

Project Viewer Type – Changes how you create and navigate the Project Viewer Dialog. Simple View displays a list of Project Names. Tree View displays a structure list of Project Names categorized by a user-defined category structure.


Preferences

Use the drop-down menu to select the preference profile to use. Preferences determine the custom tool strip used by the program. Refer to the custom tool strip section for additional information.



Project Material Sync Type

Use the drop-down menu to select how to synchronize your project materials from the two options available to you: 


Sync Current Project- The software will sync project materials with the template when a project is opened. No template changes are pushed to the project until then. 


Sync All Projects with Template Materials- Each time a material is modified in the template, the software will immediately sync the changes with all projects.

Edit Buttons

Edit Hidden Message Boxes- Selecting this option opens the Edit Hidden Message Boxes interface which allows the user to unselect messages that have previously been hidden. 





This option is only available if a message has previously been hidden. 


Edit Labels- Selecting this option opens the Label Selection interface, where one can select the current group of labels for parts, auto nesting, and products.



 

Plugins

Allow Custom Plugins- Enable this option to allow for the integration of custom plugins to your Toolbox software. 

Project Schedule

The Use Manual Completion checkbox controls the completion method of scheduled project activities. Check it to enable manual marking of completion. Leave it unchecked to allow the project to be marked as completed based on the schedule placed in the Project Properties interface. 



Report Categories

Show All Report Categories- Checking the box ON will cause Toolbox to display all the categories. This gives Toolbox Users access to modify all the reports without the use of Production. Unchecking the box will cause Toolbox to display only the categories: manufacturing reports, product labels, and placed part labels.

Part Properties


Load All Subassemblies- Enable or disable the option to load all subassemblies at once. If "Load All Subassemblies" is disabled, the software will only load subassemblies with a quantity >0, but if enabled, the software will load ALL subassemblies, regardless of whether they're currently in use or not.
