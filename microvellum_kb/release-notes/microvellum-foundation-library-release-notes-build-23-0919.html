Microvellum Foundation Library Release Notes | Build 23.0919

Additions

  1. Added new product Master Double Sided Cabinet.
  2. Added “Horizontal Handles Centered” global option to all doors. This is available for all tab pulls, handles, and knobs.
  3. Added a new DRAWRECTANGLE 2D plan token to the door subassemblies. This will turn on if the global door opening angle is set to 0. This has also addressed the issue of no 2D plan token for lift-up and sliding doors.
  4. Added "Elevation_Door_Swing_Lines" to the global and linked to the door subassemblies. This can be used to turn off the 2D elevation “swing” lines.
  5. Added global option "Use_Blind_Corner_Hinge" and linked to blind corner products.
  6. Added control type 9 prompts to the Adj Leg subassembly. This will allow the user to change to decorative legs or casters.
  7. Wall and floor lines in 2D cross sections are now globally controlled. Added two global variables:
    1. “Section Add Floor”
    2. “Section Add Wall”
  8. Added global variables to control the rear corner notch and chamfer on Corner Cabinets:
    1. “Base Corner Rear Options”
    2. “Tall Corner Rear Options”
    3. “Upper Corner Rear Options”
    4. “Notched Corner Width”
    5. “Notched Corner Depth”
    6. “Chamfered Corner Width”
    7. “Chamfered Corner Angle”
  9. Face Frame Attachment options have been added to the globals and have been linked to machine tokens in the Face Frame products to attach the face frame to the cabinet body. Default three options are “None”, “Dado”, & “Hardware Array”. Added 17 variables:
    1. “Face Frame Attachment Face Top”
    2. “Face Frame Attachment Face Bottom”
    3. “Face Frame Attachment Face Sides”
    4. “Face Frame Attachment Face Divisions”
    5. “Face Frame Attachment Face Shelves”
    6. “Face Frame Attachment Hardware Edge Array”
    7. “Face Frame Attachment Vertical Spacing”
    8. “Face Frame Attachment Vertical Distance From Edge”
    9. “Face Frame Attachment Vertical Threshold”
    10. “Face Frame Attachment Horizontal Spacing”
    11. “Face Frame Attachment Horizontal Distance From Edge”
    12. “Face Frame Attachment Horizontal Threshold”
    13. “Face Frame Attachment Face Top”
    14. “Face Frame Attachment Face Bottom”
    15. “Face Frame Attachment Face Sides”
    16. “Face Frame Attachment Face Divisions”
    17. “Face Frame Attachment Face Shelves”
  10. Added the ability to select to have only one hardware piece on the hardware array tokens on rails, nailers, and kick faces. This looks at the “G!Hardware_Array_Small_Part_Threshold” and column 9 of the “H!LookUpTableEdgeArray”. If there is a 1 in this column, it will only add one piece of hardware in the center of the part.
  11. Face Frame Dado tokens have been added to divisions and shelves.

Modifications

  1. The "Drawer_Stretcher_1" prompt has now been hidden in Door-Drawer Cabinets, as this is controlled by “Drawer Stretcher” combo box.
  2.  The 3D graphic drawings for the Shelves w Shelf Std product have been refactored. Removed “2D_” & “3D_” from the file names, as they are not using associative machining.
  3. Optimization reports and labels will now filter by processing station. “L” & “W” have been added to reports & part label dimensions. US displays dimensions (WxL). AU/UK display as (LxW).
  4. The prompt drawer order has been changed in the Component Drawers and the Master Cabinets so that the bottom drawer front height is now second in the calculator list. When changing the qty of drawers the calculator will no longer be disrupted.

Resolutions

  1. Corrected the handle location behavior of lift-up/dropdown doors. Also, the handle positions in the Component Door subassemblies have been linked to the Base, Tall, and Upper Door prompts so the correct prompts for the handle positions are visible.
  2. Added missing graphics for Richelieu RCS 165 hinges.
  3. F!LookUpTableSystemDrawerSlide has been updated to reflect the correct slide height from the bottom for certain Accuride runners.
  4. There was a reported issue with selecting a part to oversize: in the saw optimization processing station, enabling an oversized part would result in an inaccurate cut part size in the resultant saw optimization report, with the report failing to account for the differences in measurement between the basic measurement and the oversized measurements. This issue has been corrected but will require users to update their reports. Visit the Sawing Optimization Reports download page and import the new reports using the Database Management feature.
  5. Updated the “Drawer Stretcher” combo box prompt in the Door-Drawer Cabinets so the stretcher will turn on when “Add Lock” is enabled and “Drawer Stretcher” is set to "Lock Only".
  6. Resolved an issue in Door-Drawer Cabinets wherein the top-drawer lock location was different than Drawer Bank Top Drawer locks. Resolved several other related issues and differences.
  7. The “Internal Route” variables inside Finished Faces subassembly prompts will now unhide when a glass door is activated.
  8. Updated the Hettich Multitech Drawer to ensure it uses the correct library level hardware materials.
  9. The Component Folding Door subassembly in all folding door products has been updated to correct a Y origin issue.
