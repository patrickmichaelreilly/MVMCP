Synchronizing Sheet Sizes in the Microvellum Workbooks

During the process of customizing one’s library, it may be necessary to create formulas that reference certain aspects of the database material inventory, such as sheet sizes. To create a parametric library formula for a certain product, one may require Toolbox to update the material workbooks with the database sheet sizes. This process is relatively simple for users on Windows.

This method of updating sheet sizes should only be used in special circumstances, and when absolutely necessary. Updating sheet sizes in this manner is known to cause performance issues for the software. User databases with a high number of materials may have adverse effects on their synchronization speed. It is recommended that users intending to utilize this process only do so when their current setup has minimal materials applied. 
Figure 1: Updating sheet sizes process, demonstrated.
  1. To synchronize sheet sizes in Microvellum, one must utilize the registry editor application in Windows. 
  2. To open this application, type "regedit.exe” into the search or start bar on your computer. 
  3. Once the application is opened, navigate to the “HKEY_CURRENT_USER” path and select the “SOFTWARE” folder. 
  4. From “SOFTWARE” navigate to the “Microvellum” folder, then open “Toolbox”, and one’s current Toolbox OEM version (this will be the folder with the “R” prefix and the current OEM year number).
  5. Open the folder marked "Preferences", then within that, the folder marked "Materials".
  6. On the console, this should open a number of material preferences capable of being modified within the registry. Find the option labeled "UpdateSheetSizes", right-click on it, and select the "Modify" option. 
  7. From there, change the value data from 0 (the default) to 1. This update will synchronize the sheet sizes to your workbook.
