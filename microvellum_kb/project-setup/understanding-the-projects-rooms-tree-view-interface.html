Understanding the Projects & Rooms Tree View Interface

The Tree View is the UI that you use to open/create/manage projects and rooms. As of build 20.1.1103.640 Tree View has added multi-select capabilities, and the context menu can handle operations on multiple items that have been selected. There have also been speed improvements, as exemplified in Figure 09 below, and the addition of a pop-up display showing the last time the project was opened, see Figure 03. 

All of the options available in Tree View are accessed through the context menu (right-click). The options available in the context menu change depending on which item type is selected.

Project Tree View Context Menu

When the Projects Menu is opened, the following options are available in the context menu:


Fig. 01 – Project Tree View – Context Menu

  1. Move Selection(s) to Another Category – Selecting this option allows the project to be moved to another Category Folder within the Tree View interface. Select this option and choose the folder that the project is to be moved to in the Select Category interface. Multiple projects can be selected and moved at once.
  2. Copy Selection(s) to Another Category – Selecting this option allows the project to be copied to another Category Folder within the Tree View interface. Select this option and choose the folder that the project is to be copied to in the Select Category interface. A new Project Name needs to be entered. Multiple projects can be selected and copied to another category at once.
    3. The process of copying a project from within the Tree View interface also creates a new Project Folder within the Factory Database Folder and creates a new database record.
  3. Copy Selected Project(s) – Selecting this option allows the project to be copied within the same Category Folder within the Tree View interface. Select this option and enter a new name for the copy. Multiple projects can be selected and copied at once.
    4. Multiple projects can only be copied when they are within the same category or copied to a common category from multiple categories (Item #2 above).
  4. Rename Selected Project – Selecting this option allows for the project to be renamed.

    5. Fig. 02 – Rename Selected Project
    It is possible to rename a project that is currently open when no rooms from that project are open. When multiple users are working within the same database, the situation may arise where a user renames a project that another user currently has open. This scenario causes confusion between the database and room .dwgs until the project is restarted. To avoid this, an option has been added to Employee Permissions: Allow Project Rename.

     
    Fig. 03 - Allow Project Rename  

    Our recommendation is that only an administrative user is permitted to rename a project to avoid any possible confusion. It is this administrative user's responsibility to ensure the project is not currently opened by ANY  other users before the project is renamed. To learn more about Employee Permissions, see Employee Permissions and Software Security (Overview).  
  5. Delete Selected Project(s) – Selecting this option allows for the project to be deleted. Multiple projects can be selected and deleted at once.
    6. Deleting a project from the Tree View interface automatically deletes the Project Folder, from within the Factory Database Folder, and the Database record.
  6. Project Properties – Hovering over this option activates a fly-out status indicating if the selected project’s properties are active or inactive. Selecting this fly-out changes the status from inactive to active or vice versa.  
A Project cannot be deleted while it is open. In this case, a new project needs to be opened before the previous project can be deleted. 
Multiple projects or rooms can be selected at once using either Shift + Select for selecting a range or Ctrl + Select for selecting a random assortment. Only items from the same level of the tree can be selected, and only if all of the selected items are of the same type. Attempting to select an ineligible item in the tree, with multiple items selected, deselects all previously selected items.

Fig. 04 – Selecting Multiple Items in Tree View 

Room Tree View Context Menu

When the Rooms Menu is opened, the following options are available in the context menu:


Fig. 05 – Room Tree View – Context Menu

  1. Move Selection(s) to Another Category – Selecting this option allows the room to be moved to another Category Folder within the Tree View interface. Select this option and choose the folder that the room is to be moved to in the Select Category interface. Multiple rooms can be selected and moved at once.
  2. Copy Selection(s) to Another Category – Selecting this option allows the room to be copied to another Category Folder within the Tree View interface. Select this option and choose the folder that the room is to be copied to in the Select Category interface. A new Room Name needs to be entered. Multiple rooms can be selected and copied to another category at once. The user is given the option of copying the .dwg files along with the database record:

    3. Fig. 06 – Copy Drawing File(s)
    The copy of the room needs to be opened before the .dwg file is added to the Project Folder.
  3. Copy Selected Room(s) – Selecting this option allows the room to be copied within the same Category Folder within the Tree View interface. Select this option and enter a new name for the copy. Multiple rooms can be selected and copied at once.
    4. Multiple rooms can only be copied when they are within the same category or copied to a common category from multiple categories (Item #2 above).
  4. Rename Selected Room – Selecting this option allows for the room to be renamed.
  5. Delete Selected Room(s) – Selecting this option allows for the room to be deleted. Multiple rooms can be selected and deleted at once.
  6. Room Properties – Selecting this option opens the Room Properties interface.
A room cannot be renamed or deleted while it is opened. In this case, a new room needs to be opened before the previous room can be renamed or deleted. 

Categories Tree View Context Menu

When a Category is selected, the following options are available in the context menu:


Fig. 07 – Category Tree View – Context Menu

  1. Add New Category – Selecting this option allows for adding a new category within the category that is selected.
    2. When a new category is added, it is nested within the category that is selected.
  2. Rename Selected Category – Selecting this option allows for the category to be renamed.
  3. Delete Selected Category(ies) – Selecting this option allows for the deletion of the category AND all of its contents. Multiple categories can be selected and deleted, along with their contents, at once.

    4. Fig. 08 – Deletion Prompt

  4. Move Selection(s) to Another Category – Selecting this option allows the chosen category to be moved to another category. When a category is moved, all of the rooms and subcategories nested within it are also moved. Multiple categories can be selected and moved at once.
  5. Copy Selection(s) to Another Category – Selecting this option allows the category to be copied to another Category Folder within the Tree View interface. Select this option and choose the folder that the category is to be copied to in the Select Category interface. A new Category Name needs to be entered. Multiple categories can be selected and copied to another category at once.  
  6. Add New Project or Add New Room – New Projects and Rooms are added by choosing the category in which they are to be added and selecting this option.

Speed Improvements

The modifications made to the Tree View in recent builds have resulted in faster load times. When a database has many projects, load times naturally increase when opening the Tree View interface.
