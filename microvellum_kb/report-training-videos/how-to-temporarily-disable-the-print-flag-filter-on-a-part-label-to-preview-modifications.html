How to Temporarily Disable the Print Flag Filter on a Part Label to Preview Modifications

The Microvellum Part Labels contain functionality to limit the part labels that are printed to only those that are selected when printing from patterns. This is possible from Toolbox or Production in the Processing Center, or Alis.

To make this possible, we have defined a databand filter inside the Part Label reports. This works well for its intended purpose, but when the part label is being modified in the report designer, it prevents the rendered report from being displayed if the database print flag data has not been set. Here is the procedure to temporarily disable that filter to make it possible to preview the completed modifications. Also, see the attached video for the procedure.

  1. Open the report in the Report Designer.
  2. Make sure the report databands are being displayed and double click on the top-level databand if there are more than one. Usually, the data source for this databand is "OptimizationResults." If the blue databand headers are not displayed, activate them by clicking the File tab of the report designer, the Main tab to display the main options, then make sure the "Show Header" checkbox is checked.
  3. After you have double-clicked on the correct databand, click the Filters tab on either the left or right side depending on the version of Microvellum you are using. 
  4. Uncheck the checkbox for "Filter On" and click OK.
  5. Make the desired modifications, preview the report to make sure there are no errors that will prevent the report form opening in the future.
  6. Turn the filter back on, and save the report.
