Issue: No Data in Master Cut - Sorted by Processing Station

Issue

Any report or database that references the field PartsProcessingStations.LinkIDBatch data will fail to display data. An example of a default Microvellum report that was left inoperative by this database change is the “ Master Cut Report – Sorted by Processing Stations.”  It will not operate correctly for any work order databases that are created with build 15.6.1825 or later.

Cause

Microvellum build 15.6.1825 contains a modification to the data populated to the field PartsProcessingStations.LinkIDBatch. The LinkID data is no longer populated there but is populated to OptimizationResults.LinkIDWorkOrderBatch instead.

Resolution

  1. If you are using the Microvellum default report, you will need to download the current version from the Microvellum Help Center. See  Master Cut – Sorted by Processing  in the Microvellum Help Center to download the latest default report containing the modifications.
    1. Import that report, overwriting the existing report.
    2. If you are not asked to overwrite an existing report, the name of the new report is different. You will need to manually delete the existing report from the list displayed when you click Toolbox Setup > Design Reports > Design Manufacturing Report > Open Report. Left-click, then right-click on the report to delete and click Delete Selected Report. 
  2. If you are using a custom report, use the steps below to correct the report:
    1. Rollback Toolbox to a build before 15.6.1825.
    2. Launch Report Designer.
    3. Open the Report and select Edit the Data Source.
      1. Check the box to include the table Optimization Results in the report dataset.
      2. Uncheck the box for the table PartsProcessingStation to remove it from the report dataset.
      3. Check the box for  Is Batch Report.
    4. Open the Report and select Designer.
      1. Double-click on the databand named DataPartsProcessingStations and click the Filters tab. Remove the filter shown in figure 1.

    10. Update the name of the databand DataPartsProcessingStations to DataOptimizationResults.
    11. Update the Data Source for the newly named databand DataOptimizationResults to OptimizationResults.
    12. Preview the report and resolve ALL errors.
    13. Save and close the report.
