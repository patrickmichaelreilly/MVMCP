Overview: Configuring Biesse Winstore System

This article presents a brief overview of the Microvellum connection to the Biesse Winstore material handling system. 

First, it is important to understand that the communication with the Biesse Winstore is one-way, meaning that Microvellum transmits the material handling code to Winstore in the G-Code file header, but Winstore sends nothing back to Microvellum. 

Two items must be set up within Microvellum before processing for that transfer of the data to Winstore. 
  1. Complete the material handling code field for each material. 
    1. Add a material file handling code, as shown below, for each material. Besides the toolfile header, this data is also populated to the fields Sheets.HandingCode (when the work order is created), and PlacedSheets.HandlingCode (when the work order is processed). 

      2. Fig. 01
  2. Set the toolfile setting: "Auto Label Settings > Auto Labeling Type" to "BiesseWinstore."

    3. Fig. 02

  3. Process the work order and create G-Code. The data required by Biesse Winstore should then be available in the toolfile header data.
