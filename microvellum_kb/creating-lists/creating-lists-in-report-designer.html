Creating Lists in Report Designer

The articles in this section of the Knowledge Base involve working with Lists for display in your reports. To access the Report Designer, select Toolbox Setup > Design Reports > Design (Report Type) Report. Select New Report to design a report from scratch or New Report from Existing. Select Designer to open Report Designer.

Lists in a report are output using three bands: Data, Header, and Footer. Data is output using these bands. The basic band is the Data band. A data source is specified for each Data band. The data source is a table; in our case, it is the Microvellum Database. Every data source has data fields, the column headers. A table can be created by placing text components with references to these fields. One data source can specify a previously unknown number of rows with data. The number of rows in the specified data source determines how many times the Data band is output. When there is not enough space on one page, the second page is generated, and printing continues. Header bands are added to the top of the table that is output. The Footer band is used to output different totals for the table.

For information about adding bands to reports, see Understanding Report Bands in Report Designer.

The Count Data property can be used to print a Data band several times without specifying a data source.

It is possible to specify the number of elements in the Data band editor. Figure 02 shows the Data editor.


Fig. 03 – Data Editor

  1. A data source is not specified.
  2. The field and the number of elements for the Data band can be specified.
The Count Data property is set to zero by default. If set to 3, then the Data band is printed 3 times. This feature is often used to print empty data bands between populated data bands. It is important to remember that, in this case, the data source is not specified.
