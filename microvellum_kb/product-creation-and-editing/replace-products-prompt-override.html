Replace Products: Prompt Override

We have added the ability to skip a prompt when using the Replace Product command.  If you don't want to use the original product's prompt value, you can set an optional control type on that product's prompts page.
 
On the prompts page, the column named Use Last Value, controls whether a prompt will use the original product prompts value or use its default library value.  This column accepts two values separated by a pipe symbol “|”.  The first value is still used for the Use Last Value feature and if it contains a value of 1 it will use the value of the last product when it is added to a project.  If the cell contains a pipe symbol followed by a “1”, the Replace Product command will skip this prompt and use the default library prompt value.   Any values entered besides a 1 will result in the default value of 0|0 being used.  All the valid inputs for this cell are listed below.
 
  1. Blank Cell or “0|0” = Will NOT use Last Value when adding a new product from the library.  The Replace Product command will use the original product's prompt value. 
  1. “1|0” = Will use the Last Value when adding a new product from the library.  The Replace Product command will use the original product's prompt value. 
  1. “0|1” = Will NOT use the Last Value when adding a new product from the library.  The Replace Product command will NOT use the original product's prompt value.        
  1. “1|1” = Will use the Last Value when adding a new product from the library.  The Replace Product command will NOT use the original product's prompt value.
For a comprehensive overview of prompt control types, see Reference: Control Types and Other Prompt Properties.
