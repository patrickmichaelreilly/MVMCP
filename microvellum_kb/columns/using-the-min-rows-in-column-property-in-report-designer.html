Using the Min Rows in Column Property in Report Designer

The articles in this section of the Knowledge Base involve working with Columns for display in multi-column reports. To learn more about Columns and how to access the Report Designer, see Creating Multi-Column Reports in Report Designer. This article involves utilizing the Min Rows in Column property for use with Multi-Column Reports.


Fig. 01 – Min Rows in Column Property

When using the Down then Across mode of the Column Direction property, you may encounter a situation where there are not enough Data Rows to output evenly in a report. In some visualization cases, it may behoove you to NOT distribute Data Rows equally across all columns in a report.


Fig. 02 – Report Example NOT Using Min Rows in Column Property


The Min Rows in Column property is used to set the smallest number of rows in the first column of your multi-column report.

This property is set to zero by default. At zero, there is no minimum number of data rows.

When the value of this property is more than zero, then this number is used to specify the number of rows that are output in the first column. In Figure 03, the Min Rows in Column property has been set to 10 for the Subassemblies list, and 5 for the Products list.


Fig. 03 – Report Example Using Min Rows in Column Property
