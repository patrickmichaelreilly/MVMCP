Issue: Error Message "Common Language Runtime Detected an Invalid Program"

Issue:

When you start a Microvellum program, an error message is displayed that says, "Common Language Runtime detected an invalid program." 

Solution:

The value for the registry setting called LastConfig in the ApplicationSettings folder is not set to a valid configuration for the software.  You will have to manually change this value to a valid configuration on your system:

Check your configurations:
  1. Go into the registry by clicking on the start, or home button and in the search programs and files area, type in regedit and hit Enter on your keyboard.
  2. Expand the configurations in your registry and locate the configuration this station should be using for Microvellum Data. (“HKEY_CURRENT_USER\Software\Microvellum\Toolbox\R7.0\Toolbox-x001:409(or Toolbox- Standard)\Configurations).

(If this path uses a mapped drive, you may just have to make sure the user is connected to it. For more information on mapped drives see this Microsoft KB article:  http://support.microsoft.com/kb/308582 ).

Check your Application Settings:
  1. Go into the registry by clicking on the start, or home button and in the search programs and files area, type in regedit and hit Enter on your keyboard.
  2. In the registry browse to “HKEY_CURRENT_USER\Software\Microvellum\Toolbox\R7.0\Toolbox-9001:409\ApplicationSettings”. This path might vary slightly depending on the build of Toolbox that is currently installed.
  3. Find LastConfig in the ApplicationSettings part of the key entry.
  4. Make sure the LastConfig shows the right configuration, if not, double click and type in the EXACT name of the configuration to be used.
  5. Launch Toolbox.
