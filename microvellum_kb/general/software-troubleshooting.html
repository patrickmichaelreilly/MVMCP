Software Troubleshooting

This article lists a few of the hiccups our users have encountered while trying to install or launch our software and provides a tested solution for each.  If you are having trouble getting your software to cooperate and don't see a solution listed here, be sure to visit our Community where you'll find detailed discussions encompassing a wide range of topics.

Toolbox Installation Errors 

Issue: 

While installing any Microvellum product the user gets error code 1603 or gets the error message “Cannot find Setup.dll”.

Resolution:

Resolve this issue by following the steps in the following article:  Preparing to Install Microvellum Software .

“Path is not Valid” Error 

Issue:  

While booting up any Microvellum product the user gets the error message: “Path is not Valid”.

Resolution: 

  1. Close any open Microvellum products.
  2. Open File Explorer and path to your current configuration: C:\Users\<username>\AppData\Local\Microvellum\Common Microvellum Data.
  3. Open your current configuration and copy the path.
  4. Open the Registry by clicking the Windows button and typing “RegEdit”.
  5. Browse to your current configuration: HKEY_CURRENT_USER > SOFTWARE > Microvellum > Toolbox > R(Version) > Toolbox > Configurations.
  6. Select your current configuration and double-click the string named “PathToMicrovellumData.
  7. Replace the “Value Data” with the path that you copied.
  8. Close the Registry Editor and restart Toolbox. 

Error Decrypting Data

Issue:  

When opening a room, an error message pops up that says “Error Decrypting Data” and the room will not open.

Resolution: 

  1. Close down Toolbox, then reopen it.
  2. Open any drawing. 
  3. In the command bar, type “RECOVERAUTO", set to "1".           
  4. Restart the program. 
  5. You can now reopen the room and proceed. The program will now Auto Recover the drawing.
