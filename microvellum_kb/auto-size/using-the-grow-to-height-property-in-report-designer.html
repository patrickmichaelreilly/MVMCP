Using the Grow to Height Property in Report Designer

The articles in this section of the Knowledge Base involve working with Auto-Size properties in Report Designer. To learn additional information regarding Auto-Sizing properties, see Understanding Auto-Sizing Properties in Report Designer.

This article is regarding applying the Grow to Height property.

Usually, you have more than one component on a band, as in Figure 01 below:


Fig. 01 – Multiple Components on a Data Band

Rendering the report may cause some of the component’s heights to change automatically to suit the size of the contents therein. This automatic changing of height can result in unwanted breaks in the layout, as seen in Figure 02.


Fig. 02 – Unwanted Breaks in the Layout

These unwanted breaks can be avoided by binding the bottom border of a component to the lower border of the band or container in which the component is placed. This binding can be accomplished using the Grow to Height property.

Grow to Height Property


Fig. 03 – Grow to Height Property

When set to true, the Grow to Height property binds the bottom border of all components that do not change size to the bottom border of the band or container in which they are placed.

The Grow to Height property must be applied to the components within the container, not the container itself. It can be applied to all of the components within a container or band.

This gives a consistent and much better-looking result, as shown in Figure 04 below:


Fig. 04 – Grow to Height Property Applied to the Components in a Band

The Grow to Height property is set to false by default.
